swift-encoder

Year Q4 2016
Languages Shell
License MIT
Source Code Github
Resources FFmpeg Docs

A fire-and-forget shell script that encodes multiple video files with ffmpeg.

Function

The purpose of swift-ffmpeg is to simplify and automate the encoding of multiple video files. Once executed by the user, the script will automatically batch encode video files in a given folder. Encoding options are easily accessable by editing the variables within swift-encoder.sh.

Backstory

I encode piles and piles of video for archival purposes and needed a way to futher my addicton by making automating the process. Looking around on the interwebz brought me to a project called h265ize. It's built off of NodeJS. (Insert light-hearted joke about hipster-based framework here.)

h265ize isn't terrible. In fact it's quite good. However it always automatically cropped my 1080p content down to 1792p for whatever reason. It's fixed now, which probably makes my humble shell script obsolete. If you don't like hipstery Javascript framworks then I guess my script still serves a purpose.

Development

There was a problem to be solved and I was gonna be the one to solve it. I planned on employing the powers of FFmpeg to encode my videos and using a shell script to automate the process.

That still leaves one problem. I had no idea how to make shell scripts and I've only ever copy and pasted FFmpeg commands online.

After hours of looking things up on Ask Ubuntu and Stack Overflow reading FFmpeg documentation and skimming some shell scripting tutorials, I was finally able to (mostly) complete my vision. The script still remains unfinished but it has enough functionally in it at the moment for me to call it a beta.
